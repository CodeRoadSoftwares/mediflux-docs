Webhooks allow MediFlux to notify your application in real time when events occur in the platform. Instead of polling the API for changes, your application receives HTTP POST requests at a URL you specify whenever a subscribed event fires.

Webhooks are essential for building integrations that need to react to events as they happen — such as syncing sales data to an external accounting system, triggering notifications when inventory runs low, or updating a customer portal when a prescription is ready.

## How Webhooks Work

1. You register a webhook endpoint URL and subscribe to specific event types
2. When a subscribed event occurs in MediFlux, the platform sends an HTTP POST request to your endpoint
3. Your endpoint processes the event and returns a `200` status code to acknowledge receipt
4. MediFlux retries failed deliveries with exponential backoff

## Setting Up a Webhook

### Via the Dashboard

Navigate to **Settings → API → Webhooks** and click **Create Webhook**. Provide your endpoint URL, select the events you want to subscribe to, and save.

### Via the API

```typescript title="Example: Create a webhook subscription"
const webhook = await fetch("https://api.mediflux.io/v1/webhooks", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    Authorization: "Bearer your-api-key",
  },
  body: JSON.stringify({
    url: "https://yourapp.com/webhooks/mediflux",
    events: [
      "sale.completed",
      "sale.voided",
      "inventory.low_stock",
      "purchase_order.received",
    ],
    secret: "whsec_your_webhook_signing_secret",
    metadata: {
      environment: "production",
      integration: "erp_sync",
    },
  }),
});

const data = await webhook.json();
console.log("Webhook ID:", data.id);
// Output: Webhook ID: wh_abc123
```

## Available Events

MediFlux supports webhooks for the following event categories:

### Sales Events

| Event                | Trigger                                       |
| -------------------- | --------------------------------------------- |
| `sale.created`       | A new sale transaction is initiated            |
| `sale.completed`     | A sale is finalized with payment               |
| `sale.voided`        | A sale is voided before end of day             |
| `return.created`     | A return is initiated                          |
| `return.completed`   | A return is processed and refund issued        |

### Inventory Events

| Event                      | Trigger                                   |
| -------------------------- | ----------------------------------------- |
| `inventory.low_stock`      | Stock falls below the reorder point       |
| `inventory.out_of_stock`   | Stock reaches zero                        |
| `inventory.expiring_soon`  | A batch is approaching its expiry date    |
| `inventory.adjusted`       | Manual inventory adjustment recorded      |

### Purchase Events

| Event                         | Trigger                                |
| ----------------------------- | -------------------------------------- |
| `purchase_order.created`      | A new PO is created                    |
| `purchase_order.submitted`    | A PO is sent to the supplier           |
| `purchase_order.received`     | Inventory is received against a PO     |
| `purchase_order.cancelled`    | A PO is cancelled                      |

### Accounting Events

| Event                    | Trigger                                    |
| ------------------------ | ------------------------------------------ |
| `journal_entry.created`  | A new journal entry is posted              |
| `payment.received`       | A customer payment is recorded             |
| `payment.sent`           | A supplier payment is processed            |

## Verifying Webhook Signatures

Every webhook delivery is signed with your webhook secret so you can verify that requests are genuinely from MediFlux. The signature is included in the `X-MediFlux-Signature` header.

<Tabs items={["Node.js", "Python", "Go"]}>
  <Tab value="Node.js">
    ```typescript title="Verify webhook signature (Node.js)"
    import crypto from "crypto";

    function verifyWebhookSignature(
      payload: string,
      signature: string,
      secret: string
    ): boolean {
      const expected = crypto
        .createHmac("sha256", secret)
        .update(payload)
        .digest("hex");

      return crypto.timingSafeEqual(
        Buffer.from(signature),
        Buffer.from(`sha256=${expected}`)
      );
    }

    // In your webhook handler
    app.post("/webhooks/mediflux", (req, res) => {
      const signature = req.headers["x-mediflux-signature"];
      const isValid = verifyWebhookSignature(
        JSON.stringify(req.body),
        signature,
        process.env.WEBHOOK_SECRET
      );

      if (!isValid) {
        return res.status(401).json({ error: "Invalid signature" });
      }

      // Process the event
      const event = req.body;
      console.log("Received event:", event.type);

      res.status(200).json({ received: true });
    });
    ```
  </Tab>

  <Tab value="Python">
    ```python title="Verify webhook signature (Python)"
    import hmac
    import hashlib
    from flask import Flask, request, jsonify

    app = Flask(__name__)

    def verify_signature(payload: bytes, signature: str, secret: str) -> bool:
        expected = hmac.new(
            secret.encode("utf-8"),
            payload,
            hashlib.sha256
        ).hexdigest()
        return hmac.compare_digest(signature, f"sha256={expected}")

    @app.route("/webhooks/mediflux", methods=["POST"])
    def handle_webhook():
        signature = request.headers.get("X-MediFlux-Signature", "")
        is_valid = verify_signature(
            request.data,
            signature,
            os.environ["WEBHOOK_SECRET"]
        )

        if not is_valid:
            return jsonify({"error": "Invalid signature"}), 401

        event = request.json
        print(f"Received event: {event['type']}")

        return jsonify({"received": True}), 200
    ```
  </Tab>

  <Tab value="Go">
    ```go title="Verify webhook signature (Go)"
    package main

    import (
        "crypto/hmac"
        "crypto/sha256"
        "encoding/hex"
        "fmt"
        "io"
        "net/http"
    )

    func verifySignature(payload []byte, signature, secret string) bool {
        mac := hmac.New(sha256.New, []byte(secret))
        mac.Write(payload)
        expected := "sha256=" + hex.EncodeToString(mac.Sum(nil))
        return hmac.Equal([]byte(signature), []byte(expected))
    }

    func webhookHandler(w http.ResponseWriter, r *http.Request) {
        body, _ := io.ReadAll(r.Body)
        signature := r.Header.Get("X-MediFlux-Signature")

        if !verifySignature(body, signature, webhookSecret) {
            http.Error(w, "Invalid signature", http.StatusUnauthorized)
            return
        }

        fmt.Println("Webhook received and verified")
        w.WriteHeader(http.StatusOK)
        w.Write([]byte(`{"received": true}`))
    }
    ```
  </Tab>
</Tabs>

<Callout type="warning">
  Always verify webhook signatures before processing events. Without verification, an attacker could send forged webhook payloads to your endpoint.
</Callout>

## Retry Policy

If your endpoint returns a non-2xx status code or fails to respond within 30 seconds, MediFlux retries the delivery with exponential backoff:

| Attempt | Delay After Previous |
| ------- | -------------------- |
| 1       | Immediate            |
| 2       | 1 minute             |
| 3       | 5 minutes            |
| 4       | 30 minutes           |
| 5       | 2 hours              |
| 6       | 8 hours              |
| 7       | 24 hours             |

After 7 failed attempts, the delivery is marked as failed and the webhook is flagged for review. MediFlux sends a notification to the organization admins when a webhook endpoint consistently fails.

<Callout type="info">
  You can view delivery logs and retry failed deliveries manually from the **Settings → API → Webhooks** dashboard. Each delivery log includes the request payload, response status, and response body for debugging.
</Callout>

## Best Practices

- **Respond quickly** — Return a `200` status code as fast as possible. Process the event asynchronously if your handler involves heavy computation.
- **Handle duplicates** — Use the event `id` field to deduplicate events. MediFlux may deliver the same event more than once during retries.
- **Use HTTPS** — MediFlux only delivers webhooks to HTTPS endpoints. HTTP URLs are rejected during registration.
- **Monitor delivery health** — Check the webhook dashboard regularly for failed deliveries that may indicate issues with your endpoint.
- **Implement event filtering** — Subscribe only to the events your integration needs. This reduces unnecessary network traffic and processing load.
