The MediFlux API is a RESTful JSON API that gives you programmatic access to every feature of the platform. Whether you're building a custom integration, automating workflows, or connecting MediFlux to your existing systems, the API provides a consistent and well-documented interface.

All API endpoints use HTTPS and return JSON responses. Authentication is handled via API keys or OAuth 2.0 tokens, and every request is rate-limited to ensure fair usage across all clients.

## Base URL

All API requests are made to the following base URL:

```
https://api.mediflux.io/v1
```

For self-hosted installations, replace the domain with your instance's hostname:

```
https://your-instance.example.com/api/v1
```

## Quick Start

Here's a minimal example to verify your API connection:

```typescript title="Test API connection"
const response = await fetch("https://api.mediflux.io/v1/health", {
  headers: {
    Authorization: "Bearer your-api-key",
  },
});

const data = await response.json();
console.log(data);
// { "status": "healthy", "version": "2.4.1" }
```

## Available Endpoints

The API is organized around the platform's core resources:

| Resource            | Endpoint                    | Description                              |
| ------------------- | --------------------------- | ---------------------------------------- |
| Organizations       | `/v1/organizations`         | Manage organizations and settings        |
| Locations           | `/v1/locations`             | Pharmacy locations and configuration     |
| Users               | `/v1/users`                 | User accounts and permissions            |
| Products            | `/v1/products`              | Product catalog and pricing              |
| Inventory           | `/v1/inventory`             | Stock levels, batches, and adjustments   |
| Sales               | `/v1/sales`                 | Sales transactions and receipts          |
| Purchase Orders     | `/v1/purchase-orders`       | Procurement and supplier orders          |
| Suppliers           | `/v1/suppliers`             | Supplier directory and contacts          |
| Accounts            | `/v1/accounts`              | Chart of accounts                        |
| Ledger Entries      | `/v1/ledger/entries`        | Journal entries and transactions         |
| Reports             | `/v1/reports`               | Financial and operational reports        |
| Webhooks            | `/v1/webhooks`              | Webhook subscriptions and delivery logs  |

## Request Format

All write operations (POST, PUT, PATCH) accept JSON request bodies:

```typescript title="Standard request format"
const response = await fetch("https://api.mediflux.io/v1/products", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    Authorization: "Bearer your-api-key",
    "X-Location-Id": "loc_xyz789",
    "X-Idempotency-Key": "unique-request-id",
  },
  body: JSON.stringify({
    name: "Amoxicillin 500mg Capsules",
    sku: "AMX-500-30CT",
    category: "prescription",
    price: 55.0,
    requires_prescription: true,
  }),
});
```

<Callout type="info">
  Use the `X-Idempotency-Key` header on write operations to prevent duplicate processing. MediFlux stores idempotency keys for 24 hours and returns the original response for duplicate requests.
</Callout>

## Response Format

All responses follow a consistent structure:

```json title="Success response"
{
  "data": { },
  "meta": {
    "request_id": "req_abc123",
    "timestamp": "2024-01-15T14:30:00Z"
  }
}
```

```json title="Error response"
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "The 'name' field is required.",
    "details": [
      {
        "field": "name",
        "message": "This field cannot be empty."
      }
    ]
  },
  "meta": {
    "request_id": "req_abc124",
    "timestamp": "2024-01-15T14:30:01Z"
  }
}
```

## Pagination

List endpoints return paginated results. Use `page` and `per_page` query parameters to navigate:

```
GET /v1/products?page=2&per_page=50
```

Paginated responses include metadata:

```json title="Pagination metadata"
{
  "data": [],
  "pagination": {
    "page": 2,
    "per_page": 50,
    "total": 1247,
    "total_pages": 25
  }
}
```

## Rate Limiting

API requests are rate-limited per API key. Current limits are included in response headers:

| Header                  | Description                          |
| ----------------------- | ------------------------------------ |
| `X-RateLimit-Limit`     | Maximum requests per window          |
| `X-RateLimit-Remaining` | Remaining requests in current window |
| `X-RateLimit-Reset`     | Unix timestamp when the window resets|

<Callout type="warning">
  Exceeding the rate limit returns a `429 Too Many Requests` response. Implement exponential backoff in your integration to handle rate limit errors gracefully.
</Callout>
