Ledgers are the foundational records of your pharmacy's financial history. MediFlux maintains a general ledger that records every financial transaction, along with subsidiary ledgers that provide detailed breakdowns for key account categories like accounts receivable, accounts payable, and inventory.

Understanding how ledgers work in MediFlux helps you maintain accurate financial records, troubleshoot discrepancies, and generate meaningful financial reports.

## General Ledger

The general ledger (GL) is the master record of all financial transactions in your pharmacy. Every journal entry — whether generated automatically from a sale or entered manually — posts to the general ledger.

```typescript title="Example: Query general ledger entries"
const entries = await fetch("https://api.mediflux.io/v1/ledger/entries", {
  method: "GET",
  headers: { Authorization: "Bearer your-api-key" },
});

const data = await entries.json();
```

## Subsidiary Ledgers

Subsidiary ledgers provide detailed transaction histories for specific account categories. They break down the summarized balances in the general ledger into individual records.

### Accounts Receivable Ledger

Tracks amounts owed to your pharmacy by customers and insurance companies:

```typescript title="Example: Query AR ledger"
const arLedger = await fetch("https://api.mediflux.io/v1/ledger/receivables", {
  headers: { Authorization: "Bearer your-api-key" },
});

const receivables = await arLedger.json();
// Returns individual receivable records with aging information
```

| Aging Bucket  | Description             | Action Required          |
| ------------- | ----------------------- | ------------------------ |
| Current       | 0–30 days outstanding   | Normal processing        |
| 31–60 days    | Mildly overdue          | Send reminder            |
| 61–90 days    | Significantly overdue   | Escalate to collections  |
| 90+ days      | Severely overdue        | Review for write-off     |

### Accounts Payable Ledger

Tracks amounts your pharmacy owes to suppliers:

```typescript title="Example: Query AP ledger"
const apLedger = await fetch("https://api.mediflux.io/v1/ledger/payables", {
  headers: { Authorization: "Bearer your-api-key" },
});

const payables = await apLedger.json();
```

<Callout type="warning">
  Review your accounts payable aging report regularly to avoid late payment fees and maintain good supplier relationships. MediFlux can send automated payment reminders before invoices reach their due date.
</Callout>

### Inventory Ledger

The inventory ledger tracks the cost value of your pharmacy's stock at the batch level, including purchase costs, adjustments, and cost of goods sold calculations.

MediFlux supports three inventory costing methods:

| Method                  | Description                                                  | Best For                     |
| ----------------------- | ------------------------------------------------------------ | ---------------------------- |
| FIFO (First In, First Out) | Oldest inventory costs are expensed first                  | Pharmaceuticals (recommended)|
| Weighted Average        | Average cost across all batches                              | High-volume OTC products     |
| Specific Identification | Each unit tracked at its actual purchase cost                | High-value medical devices   |

<Callout type="info">
  FIFO is the recommended costing method for pharmacies because it aligns with the physical flow of pharmaceutical products — you should always dispense the oldest stock first to minimize expiry waste.
</Callout>

## Manual Journal Entries

While most entries are generated automatically, MediFlux also supports manual journal entries for transactions that don't originate from the Pharmacy module — such as recording rent payments, payroll, or year-end adjustments.

```typescript title="Example: Create a manual journal entry"
const entry = await fetch("https://api.mediflux.io/v1/ledger/entries", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    Authorization: "Bearer your-api-key",
  },
  body: JSON.stringify({
    date: "2024-01-31",
    description: "January rent payment - Downtown location",
    lines: [
      { account_code: "6200", debit: 3500.0, credit: 0 },
      { account_code: "1100", debit: 0, credit: 3500.0 },
    ],
  }),
});
```

<Callout type="warning">
  Manual journal entries require appropriate permissions (`accounting.journal.create`). All manual entries are flagged in the audit log for review during financial audits.
</Callout>

## Reconciliation

MediFlux provides bank reconciliation tools to match your ledger entries against bank statements. Import bank transactions via CSV or connect directly through supported banking integrations.

The reconciliation workflow identifies matched transactions, suggests matches for unmatched items, and flags discrepancies for manual review. Regular reconciliation ensures your books accurately reflect your actual cash position.
