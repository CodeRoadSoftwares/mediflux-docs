MediFlux supports multiple deployment methods to fit your infrastructure requirements. Whether you prefer containerized deployments with Docker, a traditional manual installation, or a fully managed cloud instance, this guide covers each approach in detail.

## System Requirements

Before installing MediFlux, ensure your environment meets the following minimum requirements:

| Component        | Minimum              | Recommended           |
| ---------------- | -------------------- | --------------------- |
| CPU              | 2 cores              | 4+ cores              |
| RAM              | 4 GB                 | 8+ GB                 |
| Disk Space       | 20 GB                | 50+ GB SSD            |
| Operating System | Ubuntu 20.04+ / RHEL 8+ | Ubuntu 22.04 LTS   |
| PostgreSQL       | 14.0                 | 16.0+                 |
| Node.js          | 18.0                 | 20.0+ LTS             |
| Redis            | 6.0                  | 7.0+                  |

<Callout type="info">
  For production deployments serving more than 50 concurrent users, we recommend the higher-end specifications along with a dedicated database server.
</Callout>

## Installation Methods

<Tabs items={["Docker", "Manual", "Cloud"]}>
  <Tab value="Docker">
    Docker is the recommended installation method for most deployments. It handles all dependencies automatically and provides consistent behavior across environments.

    ### Prerequisites

    - Docker Engine 24.0 or later
    - Docker Compose v2.20 or later

    ### Step 1: Pull the MediFlux Image

    ```bash title="Pull the latest stable image"
    docker pull mediflux/mediflux:latest
    ```

    ### Step 2: Create a Docker Compose File

    ```yaml title="docker-compose.yml"
    version: "3.9"

    services:
      mediflux:
        image: mediflux/mediflux:latest
        ports:
          - "3000:3000"
        environment:
          DATABASE_URL: postgresql://mediflux:password@db:5432/mediflux
          REDIS_URL: redis://redis:6379
          MEDIFLUX_SECRET_KEY: ${MEDIFLUX_SECRET_KEY}
          MEDIFLUX_ENV: production
        depends_on:
          db:
            condition: service_healthy
          redis:
            condition: service_started
        volumes:
          - mediflux_uploads:/app/uploads

      db:
        image: postgres:16-alpine
        environment:
          POSTGRES_USER: mediflux
          POSTGRES_PASSWORD: password
          POSTGRES_DB: mediflux
        volumes:
          - mediflux_db:/var/lib/postgresql/data
        healthcheck:
          test: ["CMD-SHELL", "pg_isready -U mediflux"]
          interval: 5s
          timeout: 5s
          retries: 5

      redis:
        image: redis:7-alpine
        volumes:
          - mediflux_redis:/data

    volumes:
      mediflux_db:
      mediflux_redis:
      mediflux_uploads:
    ```

    ### Step 3: Start the Services

    ```bash title="Launch MediFlux"
    docker compose up -d
    ```

    MediFlux will be available at `http://localhost:3000` once all services are healthy.

    <Callout type="warning">
      Change the default database password before deploying to production. Use Docker secrets or an environment file to manage sensitive credentials.
    </Callout>
  </Tab>

  <Tab value="Manual">
    Manual installation gives you full control over the deployment and is suitable for environments where Docker is not available.

    ### Step 1: Install Dependencies

    ```bash title="Ubuntu / Debian"
    sudo apt update
    sudo apt install -y postgresql-16 redis-server nodejs npm
    ```

    ```bash title="RHEL / CentOS"
    sudo dnf install -y postgresql16-server redis nodejs npm
    sudo postgresql-setup --initdb
    sudo systemctl enable --now postgresql redis
    ```

    ### Step 2: Create the Database

    ```sql title="PostgreSQL setup"
    CREATE USER mediflux WITH PASSWORD 'your-secure-password';
    CREATE DATABASE mediflux OWNER mediflux;
    GRANT ALL PRIVILEGES ON DATABASE mediflux TO mediflux;
    ```

    ### Step 3: Install MediFlux

    ```bash title="Install from npm"
    npm install -g @mediflux/server@latest
    ```

    ### Step 4: Initialize the Database

    ```bash title="Run database migrations"
    mediflux db migrate --latest
    mediflux db seed --initial
    ```

    ### Step 5: Start the Server

    ```bash title="Start MediFlux"
    mediflux start --port 3000
    ```

    <Callout type="info">
      For production deployments, use a process manager like PM2 or systemd to keep MediFlux running and handle automatic restarts.
    </Callout>
  </Tab>

  <Tab value="Cloud">

    ```bash title="Authenticate with MediFlux Cloud"
    mediflux auth login --cloud
    mediflux config set endpoint https://your-org.cloud.mediflux.io
    ```

## Verifying the Installation

After installation, verify that MediFlux is running correctly:

```bash title="Health check"
curl http://localhost:3000/api/health
```

You should receive a response like:

```json title="Expected response"
{
  "status": "healthy",
  "version": "2.4.1",
  "database": "connected",
  "redis": "connected",
  "uptime": "12s"
}
```
